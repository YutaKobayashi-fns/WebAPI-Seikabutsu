## 成果物(タスク管理機能)の機能概要  
作成者： 小林 祐太
___  

◆ 当該WebAPIについて  
　 新人研修カリキュラムの 02_C# WPF開発/03_Webアプリケーション に関する成果物として、  
　 DBに対しての基本的なCRUD機能を要した最小WebAPIを作成しました。  
___  
◆ 実装した機能についての説明  
  ① Create（生成）に関する機能  
  - タスク追加機能  
 ⇒ Swagger上で入力されたタスク情報（タスク名称・タスク詳細）をタスク情報管理DBに格納します。  
 ```
 　   ※ 作成対象の項目  
　　 1. タスク名称(入力値)  
　　 2. タスク詳細  
　　　(有効値入力時：入力値格納　無効値入力時：固定値格納 [Non input details...] )  
　　 3. 作成年月日  
　　　(作成処理完了時の年月日時分秒 [yyyy/MM/dd HH:mm:ss] )  
　　 4. 更新年月日  
　　　(年月日時分秒の初期値 [0000/00/00 00:00:00] )  
```
  ___  

  ② Read（読み取り）に関する機能  
  - タスク一覧表示機能  
 ⇒ タスク情報管理DBに格納されている全タスク情報を取得して表示します。
  
  - タスク表示機能(ID指定)  
 ⇒ タスク情報管理DBに格納されているタスク情報のうち、Swagger上で入力されたIDに該当する  
 　 タスク情報を取得して表示します。  

  - タスク表示機能(キーワード指定)  
 ⇒ タスク情報管理DBに格納されているタスク情報のうち、Swagger上で入力されたキーワードに該当する  
 　 情報（タスク名称・タスク詳細）を保有するタスク情報を取得して表示します。  

  - タスク表示機能(年月日指定)  
 ⇒ タスク情報管理DBに格納されているタスク情報のうち、Swagger上で入力された年月日に該当する  
 　 情報（作成年月日・更新年月日）を保有するタスク情報を取得して表示します。  
　 ※入力フォーマット：[ yyyy/MM/dd ]  
  ___  

  ③ Update（更新）に関する機能  
  - タスク更新機能(ID指定)  
 ⇒ タスク情報管理DBに格納されているタスク情報のうち、Swagger上で入力されたIDに該当する  
 　 タスク情報を取得して更新処理を行います。 
 ``` 
　   ※ 更新対象の項目  
　　 1. タスク名称(入力値)  
　　 2. タスク詳細  
　　　(有効値入力時：入力値格納　無効値入力時：固定値格納 [Non input details...] )  
　　 3. 作成年月日  
　　　(更新処理時に作成年月日登録なしの場合のみ初期値設定 [0000/00/00 00:00:00] )  
　　 4. 更新年月日  
　　　(更新処理完了時の年月日時分秒 [yyyy/MM/dd HH:mm:ss])  
```

  - タスク作成年月日更新機能(ID指定) 
 ⇒ タスク情報管理DBに格納されているタスク情報のうち、Swagger上で入力されたIDに該当する  
 　 タスク情報を取得して作成年月日の更新処理を行います。  
　　ただし、下記条件のいずれかを満たしている場合のみ更新可能としました。  
　　・ 作成年月日の登録がない場合 (スペースのみ・空文字・null値)  
　　・ 作成年月日が初期値の場合 ( [0000/00/00 00:00:00] )  
　　・ 作成年月日の登録フォーマットが異なる場合 ( [yyyy/MM/dd HH:mm:ss]以外 )  
```
　 ※ 更新対象の項目   
　　 1. 作成年月日  
　　　(入力値 [yyyy/MM/dd HH:mm:ss])  
　　 2. 更新年月日  
　　　(更新処理完了時の年月日時分秒 [yyyy/MM/dd HH:mm:ss])  
```
  ___  

  ④ Delete（削除）に関する機能  
  - タスク削除機能(ID指定)  
 ⇒ タスク情報管理DBに格納されているタスク情報のうち、Swagger上で入力されたIDに該当する  
 　 タスク情報を取得してDBより削除します。  

  - タスク削除機能(年月日指定)  
 ⇒ タスク情報管理DBに格納されているタスク情報のうち、Swagger上で入力された年月日(作成年月日)に  
 　 該当するタスク情報をすべて取得してDBより削除します。  
　 ※入力フォーマット：[ yyyy/MM/dd ]
